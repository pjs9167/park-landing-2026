# 내 자기소개 랜딩페이지 PRD

## 1. 목적
오프라인 업계 모임·사내외 미팅에서 종이 명함 대신 건넬 자기소개 웹페이지를
만듭니다. 카톡·문자로 링크 한 줄만 보내면 상대방이 바로 열람할 수 있어,
명함 분실·재인쇄·정보 갱신의 번거로움에서 벗어나는 것이 목적입니다.

## 2. 사용자 (페르소나)
- 이 페이지를 '전달하는 사람(나)' :
  - 이름 / 나이 / 직업 : 박준식 / 62세 / AI 컨설턴트 & 강사 (전 현대자동차 R&D 연구개발본부 팀장, 34년 경력)
  - 주로 어떤 상황에서 이 페이지를 공유하는가 : 업계 세미나 후 네트워킹 시간,
    외부 강연 자리, 기업·기관 미팅
- 이 페이지를 '받아 보는 사람(상대방)' :
  - 어떤 기기로 보는가 : 주로 스마트폰 (모임 자리에서 바로 열람)
  - 얼마나 오래 볼 것 같은가 : 10~20초
  - 무엇을 알고 싶어 하는가 : 박준식이 누구이며(이름·직책), 무슨 경력이
    있으며(2건), 어떻게 연락할 수 있는가(이메일·전화)

## 3. 기능
### 필수 기능
1. 왼쪽 1/3 너비의 네이비 사이드바에 이름·직책·이메일·전화번호가 세로로 표시됩니다.
2. 오른쪽 본문에 자기소개 문장(2줄)이 표시됩니다.
3. 경력 2건이 최신 순으로 카드 형태로 나열됩니다
   - 2026–현재 : AI 컨설턴트 & 강사 / 공공평가 심사위원
   - 1991–2024 : 현대자동차 R&D 연구개발본부 팀장
4. '이메일 보내기' 버튼이 있고, 누르면 기본 메일 앱이 열리며 수신자 주소가 자동으로 채워집니다.
5. '전화 걸기' 버튼이 있고, 누르면 전화번호가 클립보드에 복사됩니다.
6. 이메일·전화 버튼은 같은 줄에 나란히 배치됩니다.

### 선택 기능 (여유 있을 때)
1. 사이드바 상단에 원형 프로필 사진 한 장이 표시됩니다.

## 4. 제약
### 의도적으로 하지 않는 것
- 회원가입·로그인 기능은 넣지 않습니다.
- 방문자 통계·조회수·댓글·방명록은 넣지 않습니다.
- 다국어(영어·일본어 등)는 지원하지 않습니다. 한국어 단일로 만듭니다.

### 환경적 한계
- HTML·CSS 단일 파일(index.html)로 구성합니다. 서버 기능·데이터베이스는 쓰지 않습니다.
  (전화번호 클립보드 복사에 한해 최소한의 인라인 JavaScript 허용)
- 페이지는 크롬 브라우저와 스마트폰(세로 화면) 모두에서 읽혀야 합니다.
  모바일에서는 사이드바가 상단으로 자동 전환됩니다.
- 이미지는 프로필 사진 1장으로 제한합니다. (파일명: profile.jpg)
- 색상은 네이비(#1F3864) · 흰색 · 짙은 회색(#333) 세 가지만 사용합니다.

## 5. 레이아웃
- **사이드바 (좌, 33% 너비, 네이비)** : 프로필 사진(원형) → 이름 → 직책 → 구분선 → 이메일 → 전화번호
- **본문 (우, 나머지 너비)** : 자기소개 · 경력 · 연결하기 섹션이 화면 높이 내에서 균등 배분
- **섹션 구분** : 하단 실선(#e0e0e0)으로 구분, 흰 카드 박스 없음
